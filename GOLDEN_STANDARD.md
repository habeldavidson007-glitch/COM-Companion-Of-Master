## 🏛️ The Architecture: "Compiler-Lite" Pipeline

GPT's analysis was correct: We are mixing paradigms. To survive on 2GB RAM, we must commit to **LLM-as-Compiler**. The LLM does not "chat"; it transforms structured intent into an execution plan.

### The Single-Pass Pipeline (RAM Optimized)

```mermaid
graph TD
    A[User Input / Log Event] --> B[1. Signal Parser<br/>(Rule-Based, NO LLM)]
    B --> C[2. Intent Router<br/>(Rule-First, LLM Fallback)]
    C --> D{Needs Context?}
    D -- Yes --> E[3. Wiki Retriever<br/>(Enrich BEFORE LLM)]
    D -- No --> F
    E --> F[4. LLM Pass<br/>(Generate Execution Plan JSON)]
    F --> G[5. Tool Harness<br/>(Deterministic Execution)]
    G --> H[6. Core Formatter<br/>(Rule-Based Output)]
    H --> I[Final Response]
```

**Critical Rules for 2GB RAM:**
1.  **LLM Appears Exactly Once:** No pre-processing refinement, no post-processing summarization.
2.  **Retrieve Before Reason:** Wiki lookup happens *before* the LLM call so the prompt is complete.
3.  **Structured Output Only:** The LLM outputs JSON plans (`{ "action": "...", "params": {...} }`), not prose.
4.  **Signal Schema is King:** We do not pass raw text to the harness. We pass a validated signal object.

## 🚫 What We Are NOT Doing (Yet)

1.  **No UI:** No Tkinter, no Qt, no VS Code extension. Terminal output only (`print` or simple logs).
2.  **No Multi-Pass LLM:** No "refine this thought" or "summarize for the user". The first output is the final output.
3.  **No General Chat:** The system only responds to Godot project events or specific Godot queries.
4.  **No Cloud:** Everything runs locally. No API calls.

## 🧠 RAM Budget (2GB Limit)

| Component | Allocation | Strategy |
|-----------|------------|----------|
| **OS + Godot** | ~800 MB | Fixed cost |
| **COM Core (Python)** | ~200 MB | Base overhead |
| **Model (smollm2:1.7b)** | ~1.0 GB | Always loaded |
| **Model (qwen-coder:7b)** | ~4.5 GB | **LOAD ON DEMAND ONLY** |
| **Buffer** | ~0 MB | **Zero tolerance** |

**Rule:** If RAM > 90%, immediately unload `qwen-coder`. Fallback to `smollm2` or rule-based response.

---

## ✅ Success Criteria for Phase 1

1.  **Zero Crashes:** The COM process never exceeds 2GB RAM.
2.  **Accuracy:** Node path validation catches 90% of errors before runtime.
3.  **Speed:** Error explanation takes < 5 seconds (cold start) or < 2 seconds (warm).
4.  **Determinism:** Same input always produces same plan (no hallucinated actions).
