## RAM Optimization Strategy (2-3GB Constraint)

### Model Selection
| Model | RAM | Use Case | Load Strategy |
|-------|-----|----------|---------------|
| `smollm2:1.7b` | 1.2GB | Intent routing, fast replies, error explanation | Always loaded |
| `qwen2.5-coder:7b` | 4.5GB | GDScript generation, complex fixes | On-demand, unload after 10min idle |
| `tinyllama:1.1b` | 0.8GB | Emergency fallback when Godot needs RAM | Hot-swap target |

### Techniques Implemented
1. **Model Hot-Swapping** — Unload large models when RAM > 90%
2. **Response Streaming** — Token-by-token, no buffering
3. **Context Window Discipline** — Max 512 tokens for small models
4. **Query Classification First** — Rule-based before LLM (saves 1.2GB)
5. **Aggressive Idle Timeout** — Release qwen2.5-coder after 10 minutes

### Memory Budget
- **Base (idle):** 1.8GB (smollm2 + Python + OS overhead)
- **Burst (active):** 3.8GB (with qwen2.5-coder loaded)
- **Must return to base within:** 30 seconds of idle
